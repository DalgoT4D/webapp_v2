# Colors

Dalgo's color comes from **three independent sources today**. This page renders all
three together so drift is visible at a glance. Flip the **Theme** toolbar (top) to
check light vs dark.

---

## 1. Semantic theme tokens — `app/globals.css`

The real source of truth for UI. CSS variables, re-themeable, dark-mode aware.
These resolve live, so they change with the Theme toolbar.

## 2. Chart-type palette — `constants/chart-types.ts`

Hardcoded hex (Tailwind 500 series). **Ignores the `--chart-*` tokens above.**
Used by the chart-type picker / badges.

## ⚠️ Drift watch — "blue" in each system

The same conceptual color, three different hex values. This is the bug from the
original screenshot, made visible:

<SwatchGrid>
  <Swatch name="brand --primary" cssVar="--primary" />
  <Swatch name="chart bar" value={CHART_TYPE_COLORS.bar.color} />
  <Swatch name="avatar blue" value="#4285F4" />
</SwatchGrid>

**Fix (separate work):** collapse these into one source — a `tokens.json` built by
Terrazzo/Style Dictionary into both `globals.css` and a shared `palette.ts` that
`chart-types.ts` and `reports/utils.ts` import. Storybook makes the drift visible;
a single source makes it impossible.
